# Agentic Benchmark Troubleshooting

Issues encountered while building and running the Detox vs Agentic benchmark. Documented to prevent recurrence.

## Metro / Build Issues

### `yarn expo start --clear` breaks env var propagation

**Symptom:** Bundle has `METAMASK_ENVIRONMENT = "dev"` even though Metro process has `METAMASK_ENVIRONMENT=e2e`.
**Root cause:** The `--clear` flag on `yarn expo start` causes `babel-plugin-transform-inline-environment-variables` to resolve `process.env.*` with stale/default values instead of the actual process environment.
**Fix:** Don't use `--clear`. Start Metro without it:
```bash
METAMASK_ENVIRONMENT=e2e IS_TEST=true yarn expo start --port 8062
```
**Verify:** Check the served bundle:
```bash
curl -s "http://localhost:8062/index.bundle?platform=ios&dev=true&minify=false" | grep -o 'isE2E = exports.isE2E = [^;]*'
# Should show: isE2E = exports.isE2E = "true" === 'true' || "e2e" === 'e2e'
# BAD:        isE2E = exports.isE2E = undefined === 'true' || "dev" === 'e2e'
```

### Dual-port Metro breaks Expo Dev Client deep links

**Symptom:** App loads from wrong Metro port (e.g., 8062 instead of 8081), showing "Could not connect to the server" or loading the wrong bundle (dev instead of e2e).
**Root cause:** The Expo Dev Client deep link URL is built from `WATCHER_PORT` in `tests/helpers.js`. The Jest e2e config loads `.js.env` via `dotenv`, which has `WATCHER_PORT=8062`. Even though `dotenv` doesn't override existing env vars, the app's Expo Dev Client may auto-discover or cache the wrong server.
**Fix:** Use the SAME port from `.js.env` for both Detox and agentic phases. Switch Metro env between phases by killing and restarting Metro:
```bash
# Phase 1: Detox (e2e)
kill $(lsof -iTCP:8062 -sTCP:LISTEN -t)
METAMASK_ENVIRONMENT=e2e IS_TEST=true yarn expo start --port 8062 &

# Phase 2: Agentic (dev)
kill $(lsof -iTCP:8062 -sTCP:LISTEN -t)
METAMASK_ENVIRONMENT=dev yarn expo start --port 8062 &
```

### `tsconfig.json` regenerated by `yarn setup`

**Symptom:** `git status` shows `tsconfig.json` modified after `yarn setup` / `yarn a:setup:ios`.
**Root cause:** `yarn setup` strips comments from `tsconfig.json`. No functional change.
**Fix:** `git checkout tsconfig.json` to revert.

## App State Issues

### Wallet locked after Metro restart

**Symptom:** Agentic recipes fail with `wallet.unlocked: false`, route is `Login`.
**Root cause:** When Metro restarts and the app reloads, the wallet locks automatically (security feature). The app lands on the Login screen.
**Fix:** Unlock via CDP before running recipes:
```bash
node scripts/perps/agentic/cdp-bridge.js unlock <password>
```
The benchmark script reads the password from `.agent/wallet-fixture.json`.

### Fresh wallet onboarding screens block navigation

**Symptom:** After `preflight.sh --wallet-setup`, the app is stuck on `OptinMetrics` screen. Navigation commands don't work.
**Root cause:** Three onboarding modals must be dismissed before the app is usable:
1. `OptinMetrics` — press `optin-metrics-continue-button-id`
2. `MultichainAccountsIntroModal` — press `multichain-accounts-intro-modal-close-button`
3. `PredictGTMModal` — press `predict-gtm-not-now-button`

**Fix:**
```bash
CDP="node scripts/perps/agentic/cdp-bridge.js"
$CDP press-test-id optin-metrics-continue-button-id; sleep 2
$CDP press-test-id multichain-accounts-intro-modal-close-button; sleep 2
$CDP press-test-id predict-gtm-not-now-button; sleep 2
```

### Perps provider not authenticated after app restart

**Symptom:** Pre-condition `perps.ready_to_trade` fails with `isAuthenticated: false` or `no active provider`.
**Root cause:** PerpsController needs manual initialization after app restart. The init sequence matters.
**Fix:** Call in this order:
```bash
$CDP eval-async 'Engine.context.PerpsController.init().then(function(){return Engine.context.PerpsController.toggleTestnet(true)}).then(function(){return JSON.stringify({ok:true})})'
```
**Important ordering:**
- `init()` first — creates the provider
- `toggleTestnet(true)` second — switches to testnet
- Do NOT call `reconnect()` after `toggleTestnet()` — it resets the testnet flag
- `toggleTestnet()` internally reconnects with the correct network

**Verify:**
```bash
$CDP check-pre-conditions '["perps.ready_to_trade","perps.sufficient_balance"]'
# Should show: "ok": true
```

## PerpsController API Gotchas

### Limit order price param is `price`, not `limitPrice`

**Symptom:** `ORDER_LIMIT_PRICE_REQUIRED` error from `placeOrder()`.
**Root cause:** The `OrderParams` type uses `price` (not `limitPrice`) for limit order prices.
**Fix:**
```javascript
PerpsController.placeOrder({
  symbol: 'ETH',
  isBuy: true,
  orderType: 'limit',
  price: '2357',      // NOT limitPrice
  size: '0.001',
  usdAmount: '10'
})
```

### `getMarkets()` returns metadata, not prices

**Symptom:** Price field is `null` or `undefined` from `getMarkets()`.
**Root cause:** `getMarkets()` returns market metadata (symbol, szDecimals, maxLeverage). Live prices come from `getMarketDataWithPrices()`.
**Fix:**
```javascript
// Wrong:
PerpsController.getMarkets()  // returns {symbol, szDecimals, maxLeverage} — no price

// Right:
PerpsController.getMarketDataWithPrices()  // returns {symbol, price, change24h, ...}
```

### Price format needs stripping

**Symptom:** `placeOrder()` fails or uses wrong price when passing formatted price string.
**Root cause:** `getMarketDataWithPrices()` returns prices as formatted strings like `"$2,357.4"`. The `placeOrder()` API expects a plain number string like `"2357.4"`.
**Fix:** Strip `$` and `,` before passing:
```javascript
var raw = market.price;          // "$2,357.4"
var clean = raw.replace(/[$,]/g, '');  // "2357.4"
```

## Full Setup Sequence (after clean build)

```bash
# 1. Build
yarn a:setup:ios                    # dev build + wallet setup
yarn test:e2e:ios:debug:build       # Detox build

# 2. Dismiss onboarding (after first wallet setup)
CDP="IOS_SIMULATOR=mm-2 WATCHER_PORT=8062 node scripts/perps/agentic/cdp-bridge.js"
$CDP press-test-id optin-metrics-continue-button-id
$CDP press-test-id multichain-accounts-intro-modal-close-button
$CDP press-test-id predict-gtm-not-now-button

# 3. Init perps
$CDP eval-async 'Engine.context.PerpsController.init().then(function(){return Engine.context.PerpsController.toggleTestnet(true)}).then(function(){return JSON.stringify({ok:true})})'

# 4. Verify
$CDP check-pre-conditions '["perps.ready_to_trade","perps.sufficient_balance"]'

# 5. Run benchmark
bash scripts/perps/agentic/run-timing-benchmark.sh
```
